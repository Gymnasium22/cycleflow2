#### **A. Редактирование симптомов**
Пользователь должен иметь возможность:
- Изменять интенсивность уже записанного симптома

**Реализация:**
```
Добавить в useSymptoms:
- updateSymptom(id, updates) - обновление существующего симптома
- Модальное окно редактирования при клике на симптом на главной
```

#### **B. История симптомов в отдельной странице**
- Календарь с отметками о наличии симптомов

#### **C. Заметки к менструациям**

- Пример: "особенно обильно", "спазмы", и т.д.
- Отображение при просмотре истории

## 📊 Структура данных для новых функций

### Расширение таблицы `cycles`
```sql
ALTER TABLE cycles ADD COLUMN (
  end_date DATE,                    -- дата завершения менструации
  actual_period_length INT,         -- реальная длительность
  notes TEXT,                       -- примечания пользователя
  tracking_started_at TIMESTAMP     -- когда начали отслеживать
);
```

### Расширение таблицы `symptoms`
```sql
ALTER TABLE symptoms ADD COLUMN (
  notes TEXT,                       -- дополнительные примечания
  updated_at TIMESTAMP DEFAULT NOW()
);
```

### Новая таблица `trends`
```sql
CREATE TABLE trends (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES profiles(id),
  metric_type VARCHAR(50),          -- 'cycle_length', 'period_length', 'symptom_frequency'
  value FLOAT,
  calculated_at TIMESTAMP,
  period_start_date DATE,           -- для какого периода рассчитано
  period_end_date DATE
);
```

## ❓ Ответы на твои вопросы

### "Почему симптомы не отображались?"
- Они БЫЛИ сохранены в localStorage/Supabase
- Просто не было отображения их истории
- Теперь они видны в "Сегодня записанные симптомы" на главной
- Планируется добавить полную историю в отдельную страницу

### "Куда сохраняются данные?"
- Если авторизован через Telegram: **Supabase** (облако, постоянно)
- Если не авторизован: **localStorage** (браузер, теряется при очистке)
- Локальное приложение на телефоне может синхронизировать при входе

### "Почему настройки не сохранялись?"
- Они сохранялись, но без визуального подтверждения
- Теперь при нажатии "Сохранить все" появляется ✓ Сохранено
- Изменения не теряются при переходе на другую страницу
