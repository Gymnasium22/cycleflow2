# План исправлений и улучшений Cicle

## 1. Баги

### 1.1 Тёмная тема — белые теги самочувствия
- **Файл**: `src/pages/Home.jsx:234`
- **Проблема**: `bg-white` жёстко задан, а текст использует `var(--tg-theme-text-color)`. В тёмной теме Telegram текст = белый, фон = белый → невидимый текст.
- **Исправление**: Заменить `bg-white` на `bg-[var(--tg-theme-secondary-bg-color,#f3f4f6)]` и добавить `border` для контраста.

### 1.2 Настройки не сохраняются при переходе
- **Файл**: `src/pages/Settings.jsx`
- **Проблема**: Слайдеры и чекбоксы меняют только локальный `useState`. Сохранение в Supabase/localStorage происходит только при нажатии «Сохранить все». При переключении вкладки state теряется.
- **Исправление**: Добавить auto-save с `useEffect` + debounce при изменении `cycleLength`, `periodLength`, `notifyPeriod`, `notifyOvulation`.

### 1.3 Нет возможности изменить самочувствие
- **Файл**: `src/pages/Home.jsx`
- **Проблема**: У симптома есть только кнопка «×» (удалить). Нельзя изменить интенсивность, если ошибочно нажал.
- **Исправление**: При клике на тег самочувствия открывать модалку с текущим значением и возможностью выбрать новую интенсивность (или удалить). Использовать `updateSymptom` из `useSymptoms`.

### 1.4 Нет отмены «Месячные начались» / нет даты окончания
- **Файл**: `src/pages/Home.jsx`, `src/hooks/useCycles.js`
- **Проблема**: Нельзя отменить ошибочное нажатие. Нельзя отметить, что месячные закончились.
- **Исправление**: Если цикл начался сегодня — показать кнопку «Месячные закончились». Добавить поле `end_date` в `addCycle` / `updateCycle`. Или проще: показывать кнопку «Отменить» если цикл начат сегодня.

### 1.5 Тёмная тема в Calendar и Layout
- **Файлы**: `src/pages/Calendar.jsx`, `src/components/Layout.jsx`
- **Проблема**: `hover:bg-black/5` плохо выглядит в тёмной теме. Фиксированные цвета в легенде календаря могут быть нечитаемы.
- **Исправление**: Заменить на `hover:bg-[var(--tg-theme-hint-color,#d1d5db)]/20`.

## 2. Улучшения

### 2.1 Редактирование period_length в цикле
- В модалке редактирования цикла добавить слайдер для `period_length` (дней месячных).

### 2.2 Заметки к симптомам
- В модалке самочувствия добавить текстовое поле «Заметки» (уже есть в `useSymptoms.saveSymptom` через `notes`, но UI отсутствует).

### 2.3 Auto-save настроек
- Settings сохраняются автоматически при изменении любого поля (debounce 500ms для слайдеров, мгновенно для чекбоксов и языка).

### 2.4 Улучшенная аналитика
- Рассчитывать среднюю длину цикла на основе истории `cycles`, а не только `profile.cycle_length`.
- Показать среднюю длительность месячных по истории.
- Добавить график PMS (предположительно по дням перед месячными).

### 2.5 Экспорт данных
- Кнопка в Settings для экспорта циклов и симптомов в JSON (или CSV).

## 3. Хранение данных

Данные хранятся в двух местах:
- **Supabase** (PostgreSQL): `profiles`, `cycles`, `symptoms`, `settings` — для авторизованных пользователей (через Telegram WebApp auth).
- **localStorage**: `cicle_cycles`, `cicle_symptoms`, `cicle_settings` — для гостей/без авторизации (fallback mode).

Если переменные окружения `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY` не заданы, приложение работает в mock-режиме с пустыми данными.
